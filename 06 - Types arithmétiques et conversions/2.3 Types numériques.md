# Types numériques (théorie)

1)	Dressez la liste exhaustive de tous les types entiers supportés par C++
2)	Idem pour les réels
3)	Idem pour les caractères (à code non étendu)
4) Le type int est-il signé ou non signé par défaut ?
5) Le type char est-il signé ou non signé par défaut ?
6) Le domaine de définition des entiers est-il fixé par la norme ou dépend-il de l'environnement utilisé ?


<details>
<summary>Solution</summary>
1)	types entiers 

~~~cpp 
// chaque ligne correspond à un type qui peut s'écrire de diverse manières.

signed char;
signed short; short;
signed int; int; signed;
signed long int; signed long; long int; long;
signed long long int; signed long long; long long int; long long; 

unsigned char;
unsigned short;
unsigned int; unsigned;
unsigned long int; unsigned long; 
unsigned long long int; unsigned long long; 

// Note: l'ordre des mots clés est quelconque. Le type signed long long it peut aussi s'écrire
signed long long int; signed long int long; signed int long long, int signed long long; int long signed long; int long long signed; long signed long int; ... 
~~~

2) types réels

~~~cpp
float;
double;
long double;
~~~

3) types caractère 

~~~cpp 
char;
signed char;
unsigned char;
~~~

4)	Signé
5)	Dépend du compilateur utilisé
6)	Dépend de l'environnement utilisé
</details>
